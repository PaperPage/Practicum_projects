# 🧪 Practicum_projects

Репозиторий с результатами выполнения практических работ по курсу тестирования ПО.

## 📌 О проекте
В этом репозитории собраны чек-листы, тест-кейсы, таблицы классов эквивалентности и отчёты по дефектам. Основная цель — демонстрация навыков анализа требований, проектирования тестов, ручного тестирования веб- и мобильных приложений, а также проверки API.

## 📁 Структура репозитория
Файлы сгруппированы по типам задач и проектам:

- 📡 **`Тестирование API.xlsx`** — проверка новых эндпоинтов бэкенда (наборы, курьерская доставка, корзина). Валидация запросов/ответов, статус-коды, схема данных.
- 📱 **`Тестирование мобильного приложения.xlsx`** — регрессионное тестирование Android-приложения после рефакторинга. Проверка взаимодействия с ОС, прерываний, жестов, установки/обновления.
- 🛴 **`Тестирование приложения яндекс Самокат....xlsx`** — комплексная проверка сервиса аренды самокатов: веб-форма, мобильное приложение v2 и API-методы.
- 🗺️ **`Тестирование фичи.xlsx`** — тестирование новой функциональности веб-сервиса построения маршрутов: валидация полей ввода и расчёт времени/стоимости поездки.

## 📄 Что внутри файлов
Каждый `.xlsx` файл содержит структурированные артефакты тестирования:
- ✅ **Чек-листы** с атомарными проверками и статусами выполнения.
-  **Тест-кейсы** с предусловиями, шагами воспроизведения и ожидаемым результатом.
- 🐛 **Баг-репорты** с заголовками, окружением, шагами, ОР/ФР, серьёзностью и скриншотами.
- 📊 **Таблицы тест-дизайна** (классы эквивалентности, граничные значения).

## 🛠 Использованные инструменты
- **Microsoft Excel / Google Sheets** — проектирование тестов, ведение чек-листов и баг-трекинга.
- **Postman** — отправка HTTP-запросов, проверка ответов API.
- **Chrome DevTools** — анализ сетевых запросов, элементов и консоли.
- **Android Studio / Эмулятор** — запуск и тестирование мобильных сборок.
- **Figma** — сверка интерфейсов с макетами.

## 🚀 Как пользоваться
1. Откройте нужный `.xlsx` файл прямо в браузере (GitHub поддерживает предпросмотр таблиц) или скачайте его.
2. Перейдите на вкладку, соответствующую типу артефакта (`Чек-лист`, `Тест-кейсы`, `Баг-репорты`).
3. Для воспроизведения проверок используйте ссылки на тестовые стенды, указанные в файлах.
4. ⚠️ *Все материалы курсов, теория и комментарии ревьюера удалены. В репозитории только самостоятельно созданные артефакты.*
